Sincronización (equipo compartido)

Cómo activarlo (una sola vez, ~10 minutos):

  1. Ve a console.firebase.google.com y crea un proyecto gratis (con tu cuenta de Google).
  2. En el proyecto, entra a Compilación → Firestore Database → "Crear base de datos" → elige modo producción y cualquier región.
  3. En la pestaña Reglas de Firestore, reemplaza el contenido por esto y publica: 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
    ⚠ Esto permite que cualquiera con el enlace de tu proyecto lea/escriba los datos — está bien para un equipo interno, pero no publiques el archivo con esta configuración en un sitio público.
  4. Ve a Configuración del proyecto (⚙) → General, baja a "Tus apps", crea una app Web (icono </>), y copia el objeto firebaseConfig que te muestra.
  5. Pega ese objeto completo aquí abajo y da clic en "Conectar".

Cambios en tiempo real: todos ven las ediciones de los demás al instante.

Para cada persona de tu equipo (~2 minutos, los campos de repositorio ya vienen llenos):

  1. En GitHub, ve a Settings → Developer settings → Personal access tokens → Fine-grained tokens → "Generate new token".
  2. Limita el token a este repositorio y dale permiso de Contents: Read and write.
  3. Copia el token generado (empieza con github_pat_...) — solo se muestra una vez.
  4. Pégalo abajo en "Personal Access Token" y da clic en "Conectar" — el resto de los campos ya están listos.

💡 Si prefieres no generar un token por persona, también puedes darle a tu equipo directamente el mismo token que tú generaste — funciona igual, solo que no podrás revocarle el acceso a una sola persona sin afectar a las demás.

⚠ El token queda guardado en el navegador de cada persona que lo use. No lo compartas fuera de tu equipo, y si el repositorio es público, usa un token limitado solo a este repo.

Nota: esto no es instantáneo como Firebase — la app revisa cambios cada ~45 segundos y cada guardado crea un commit en tu repositorio.

Los datos se guardan como un archivo JSON en tu propio repositorio — puedes verlo y hasta revisar el historial de cambios ahí mismo.
